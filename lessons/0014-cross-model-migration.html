Lesson 14 · Claude → GPT → Gemini → 国产 · 约 25 分钟

你花了很多精力学习 Anthropic 的最佳实践。但现实是：你可能需要在不同模型间切换——成本、延迟、可用性、合规要求。 本课教你如何迁移 prompt，而不是为每个模型重写。

我们的核心发现：Anthropic 教你的大部分技巧是通用的。 你需要调整的是格式、参数和少量行为差异。

迁移策略：格式层

XML 标签 → Markdown 标题：Claude 对 XML 特别敏感，但其他模型可能更适应 markdown。

// Claude 优化版（XML）
<task>分析这份报告</task>
<context>...</context>

// 通用版（Markdown）
## 任务
分析这份报告

## 上下文
...

经验：以 markdown 版本为"通用格式"，如果需要极致效果再用 XML（仅 Claude）。

迁移策略：行为层

不同模型有不同的"性格"——这些需要在 prompt 中补偿：

模型倾向补偿策略
Claude谨慎、精确、擅长指令遵循可以更直接，需要"允许行动"的语言
GPT-4o创造性、冗长、易偏离更严格的输出约束、更频繁的格式指示
Gemini结构化、偏好列表明确要求"散文段落"而不是列表
DeepSeek / 国产指令遵循中等、对长上下文敏感更短的 prompt、更明确的格式、更频繁的自检

迁移检查清单

迁移 prompt 到新模型时，检查这些：

  1. 格式 — XML 要转成 Markdown 吗？
  2. 参数 — effort / reasoning_effort 要映射吗？
  3. System Prompt — 新模型是否有 system prompt 支持？
  4. 输出格式 — structured_output / response_format 是否需要改？
  5. 行为补偿 — 新模型需要更严的约束还是更宽松？
  6. 评测 — 用同一套评测集在两个模型上跑，对比结果

以下是一条为 Claude 优化的 prompt。请改写为一个通用版本，能在 GPT-4o 和国产模型上同样有效。

<task> 将以下英文新闻摘要翻译成中文，保持新闻语体的简洁客观。 </task> <news> The company reported a 23% increase in quarterly revenue... </news> <requirements> - 不使用"据悉"、"据了解"等套话 - 保持数字精确 - 在 <output> 标签中输出 </requirements>
提示：将 XML 标签改为 Markdown 标题即可。
## 任务
将以下英文新闻摘要翻译成中文，保持新闻语体的简洁客观。

## 新闻
The company reported a 23% increase in quarterly revenue...

## 要求
- 不使用"据悉"、"据了解"等套话
- 保持数字精确
- 不要开场白，直接输出翻译结果
