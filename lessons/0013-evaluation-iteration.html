评测与迭代

Lesson 13 · 系统化优化提示词 · 约 25 分钟

前面 12 课学了各种技巧。但没有银弹——每个任务、每个模型、每个场景都需要不同的组合。 本课教你如何系统化地评测和改进 prompt。

► 官方参考：Anthropic — Prompt engineering overview

Anthropic 强调：在开始 prompt engineering 之前，先定义成功标准和评测方法。没有评测的优化是"用感觉飞行"。

第一步：定义成功标准

在你写任何 prompt 之前，先回答："什么样的输出算好？"

维度示例标准评测方法
准确率分类正确率 > 95%用标注数据集测试
格式合规100% 合法 JSONschema 校验
完整性覆盖所有必需字段字段检查
风格无开场白，直接回答人工抽查
幻觉率虚构引用 < 2%引用验证

第二步：构建评测集

收集 10–50 个代表性输入，涵盖：

  • 典型场景 — 最常见的情况
  • 边界情况 — 空输入、极长输入、特殊字符
  • 困难案例 — 过去出过错的情况
  • 对抗样本 — 试图误导模型的输入

用 JSON 格式管理评测集：

[
  {
    "id": "tc-001",
    "input": "你们的App太难用了",
    "expected": "投诉",
    "category": "典型"
  },
  {
    "id": "tc-002",
    "input": "",
    "expected": "其他",
    "category": "边界"
  }
]

第三步：迭代循环

测试 → 分析失败 → 修改 prompt → 验证

关键规则：

  • 每次只改一个变量（不要同时改 prompt 和 effort）
  • 每次修改后在整个评测集上运行
  • 记录每次修改的效果（改进了什么、退步了什么）
  • 接受"trade-off"——提升 A 类任务可能降低 B 类任务

自评测：让 AI 评估 AI

你可以让 AI 帮你评估输出质量：

<task>
评估以下 AI 回答的质量。评分维度：
1. 准确性（是否基于提供的材料？）
2. 完整性（是否覆盖了所有需求？）
3. 格式合规（是否符合输出格式要求？）

每个维度 1–5 分，给出理由。

<prompt>
（原始 prompt）
</prompt>

<response>
（AI 的回答）
</response>
经验：AI 自评测可以快速发现明显的质量问题（格式不对、遗漏要求），但不要依赖它判断事实准确性——对事实类任务，需要人工验证。

Prompt 版本管理

把 prompt 当作代码来管理：

# prompts/classifier/v1.txt     # 初始版本
# prompts/classifier/v2.txt     # 加了 few-shot
# prompts/classifier/v3.txt     # 加了 CoT
# prompts/classifier/v4.txt     # 加了 XML 结构

并用变更记录追踪效果：

# CHANGELOG.md
## v4 - 2026-06-16
- 增加了 XML 结构，分类准确率从 89% → 94%
- 少数"中性"样本被误判为"负面"（trade-off 接受）

## v3 - 2026-06-15
- 增加了 CoT 要求，复杂案例准确率 +5%
- 简单案例速度变慢（改用 effort: low 解决）

✍️ 练习：设计一个评测计划

你要为"智能客服"系统优化 prompt。请写出你的评测计划：成功标准、评测集构成、迭代流程。

提示：包含准确率目标、评测集规模、迭代循环。
## 成功标准
- 意图分类准确率 > 95%（投诉/退换货/咨询/其他）
- 100% 使用标准话术模板（不虚构政策）
- 响应时间 < 3 秒（effort: medium）

## 评测集（30 条）
- 15 条典型场景（各类意图各 3–5 条）
- 8 条边界情况（空消息、超长消息、表情符号）
- 5 条困难案例（意图模糊、混合意图）
- 2 条对抗样本（试图诱导泄露政策）

## 迭代流程
1. 用评测集运行当前 prompt → 记录准确率
2. 分析误判样本 → 找共同模式
3. 修改 prompt（一次只改一处）
4. 重新运行评测集 → 对比结果
5. 记录 changelog

总结

  1. 先定义成功标准 — 没有目标就没有优化方向
  2. 评测集是核心资产 — 10–50 条，覆盖典型/边界/困难/对抗
  3. 一次改一个变量 — 否则你不知道什么起了作用
  4. 自评测辅助人工 — AI 可以快速检查格式和完整性
  5. 版本管理 — prompt 是代码，需要 changelog

课后行动：为你当前工作中最重要的 prompt 建立一个评测集（即使只有 10 条测试用例）。然后运行、分析、改进——你会有惊喜。
