输出控制与格式化

Lesson 4 · 让 AI 按你的要求输出 · 约 25 分钟

前三课我们学了如何描述任务、展示示例、引导推理。本课聚焦一个常被低估但极其实用的能力： 精确控制 AI 的输出格式。

很多时候 AI 的回答"内容对了但格式不对"——多了开场白、用了 markdown、该输出 JSON 却给了散文。 这些不是模型笨，是你没告诉它怎么输出。

► 官方参考：Anthropic — Output and formatting

Anthropic 发现：新模型（Opus 4.6+）默认更简洁直接，但也可能跳过你想要的总结。如果你想要什么，直接说——不要指望默认行为正好是你需要的。

原则一：说"要什么"，不说"不要什么"

这是最容易被忽略的技巧。AI 对正面指令的响应远好于负面禁令。

✗ 禁令式
不要用 markdown 格式。 不要有开场白。 不要列清单。
✓ 正面式
用流畅的散文段落回答。 直接给出核心内容，不要前缀。 用自然段落组织信息。
Anthropic 官方原话："Claude's latest models have a more concise and natural communication style... This means Claude may skip verbal summaries after tool calls, jumping directly to the next action."

原则二：输出格式指示器

用 XML 标签告诉 AI 应该用什么格式输出：

<task>
提取以下合同中的关键信息。
</task>

<output_format>
请将结果放在 <result> 标签中，每条信息一行，格式为「字段名: 值」。
</output_format>

<contract>
（合同内容）
</contract>

或者直接要求 JSON：

<task>
将以下会议记录转为结构化数据。
</task>

<output_format>
请输出 JSON 格式，字段如下：
{
  "meeting_date": "日期",
  "attendees": ["参会人列表"],
  "decisions": [{"topic": "议题", "decision": "决定"}],
  "action_items": [{"owner": "负责人", "task": "待办事项", "deadline": "截止日期"}]
}
</output_format>

原则三：prompt 风格 = 输出风格

这是 Anthropic 的经验发现：如果你在 prompt 里用了 markdown，AI 也更倾向用 markdown 输出。如果你想减少 markdown，先减少自己 prompt 里的 markdown。

AI 的输出风格会受到你 prompt 风格的强烈影响。如果你用 markdown 写 prompt，AI 也会用 markdown 回答。

你的 promptAI 倾向的回答风格
markdown 标题、列表、代码块markdown 格式回答
散文段落、自然语言散文段落回答
JSON / YAML 格式结构化数据回答
正式、书面语正式风格回答

实用技巧：如果你希望 AI 不用 markdown，先确保你的 prompt 本身没有 markdown。

原则四：前置需求声明

如果你需要特定的沟通风格或行为，在 prompt 开头明确声明。Anthropic 推荐用专门的标签包裹这些需求：

<avoid_excessive_markdown_and_bullet_points>
写报告、文档或技术分析时，用完整的段落和句子写作。
只在以下情况使用列表：(a) 真正需要列举离散项时。
用 inline code 和代码块，避免用 **bold** 和 *italics*。
</avoid_excessive_markdown_and_bullet_points>

如果你需要 AI 在完成任务后总结：

在完成涉及工具调用的任务后，请简要总结你做了什么。

✍️ 练习：重写输出控制 prompt

下面这个 prompt 要输出一份竞品分析报告，但用了禁令式控制。请重写为正面指令，并指定 XML 输出格式。

不要用 markdown。 不要列清单。 不要有开场白。 不要用 "在本文中" 这种话。 直接说重点。
提示：说"要什么"而不说"不要什么"，用 <output_format> 标签指定结构。
<task>
写一份竞品分析报告。
</task>

<style_guidelines>
用完整的散文段落写作。用自然段落和空白分隔不同主题，而不是列表。直接进入主题，不需要开场白或过渡短语。
</style_guidelines>

<output_format>
报告用 <report> 标签包裹。每个竞品分析用 <competitor> 标签分隔。
</output_format>

总结

  1. 正面指令 — 说"要什么"比"不要什么"有效得多
  2. 输出格式指示器 — 用 <output_format> 或直接给出结构模板
  3. 风格匹配 — 你的 prompt 风格决定 AI 的输出风格
  4. 前置声明 — 在开头用标签声明风格和格式要求

课后行动：找一条你常用的 prompt，把所有"不要X"换成"要用Y输出"，加上明确的输出格式指示，对比效果。
