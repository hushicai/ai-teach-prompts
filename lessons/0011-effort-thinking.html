Effort 参数与自适应思考

Lesson 11 · 控制推理深度与成本 · 约 20 分钟

Claude 4.6+ 引入了 adaptive thinking（自适应思考）——模型自动判断需要思考多深。 通过 effort 参数，你可以精确控制这个深度。

本课的核心洞察："更加努力"不总是好的。不同任务需要不同推理深度。 用错 effort 级别不仅浪费 token，还可能降低输出质量。

► 官方参考：Anthropic — Adaptive Thinking

Anthropic 发现：新模型过度思考是常见问题。如果你之前用 anti-laziness prompt（"请仔细思考"），在新模型上反而会导致性能下降。

Effort 级别速查

级别用途典型场景相对成本
low简单任务，不需要深度推理分类、格式化、简单提取低
medium默认级别，平衡质量与速度一般问答、文档分析中
high需要多步推理的复杂任务编码、数据分析、逻辑推理高
max / xhigh最高推理深度复杂 agent 任务、长链推理最高

过度思考：信号与修复

信号：AI 在简单问题上花了大量 token "思考"，回答过于冗长或过度复杂。

原因：

  • 你的 prompt 包含了 anti-laziness 语言（"请务必仔细思考"）
  • effort 级别设得太高
  • system prompt 过于复杂触发了不必要的推理

修复方法：

// 在 prompt 中限制过度思考
深思会增加延迟，只有在真正能提升回答质量时使用——通常是多步推理问题。
不确定时，直接回答。

// 选择方案后不要反复犹豫
决定方案后坚决执行。除非遇到新矛盾信息，否则不要反复评估。
在两个方案之间犹豫时，选一个做下去。

迁移：从 Extended Thinking 到 Adaptive Thinking

如果你之前用过 budget_tokens：

// 旧方式（extended thinking）
thinking: { "type": "enabled", "budget_tokens": 32000 }

// 新方式（adaptive thinking）
thinking: { "type": "adaptive" }
output_config: { "effort": "high" }

不再需要手动设 budget_tokens——adaptive thinking 自动管理推理深度，由 effort 参数控制。

跨模型提示： GPT-4o 有类似的 reasoning_effort 参数（low/medium/high）。 Gemini 通过 thinking_config 控制。 国产模型大多还没有等效参数——需要通过 prompt 手动控制。

Effort 与 Prompt 的配合

Effort 不是替代 prompt 设计的。正确的做法：

任务EffortPrompt 关键点
邮件分类（3 类）low清晰定义类别 + 2–3 个示例
文档摘要medium指定长度和焦点
代码审查high定义审查标准 + CoT 要求
复杂 agent 任务max明确的工具定义 + 状态管理
✏️ 测验 1：以下哪个任务最适合用 effort: "low"？
✍️ 练习：为不同任务分配 Effort 级别

以下 4 个任务，各应分配什么 effort 级别？写出理由。

任务 A：将一段 JSON 转为 CSV
任务 B：分析季度财报并撰写摘要
任务 C：用多步推理解数学应用题
任务 D：自主浏览多个网页并交叉验证信息 
任务 A（JSON→CSV）→ effort: low
理由：纯格式转换，不需要推理。

任务 B（财报分析）→ effort: medium
理由：需要理解数据但不需要深度多步推理。

任务 C（数学应用题）→ effort: high
理由：需要多步推理，容易出错。

任务 D（自主浏览验证）→ effort: max
理由：需要跨多源信息推理和验证，最复杂的任务。

总结

  1. Adaptive Thinking = 模型自动决定思考深度
  2. Effort 参数 low → medium → high → max 对应递增的推理深度
  3. 过度思考有害 — 移除 anti-laziness prompt，用 effort 级别代替
  4. 迁移 — budget_tokens 已废弃，改用 effort
  5. 配合 prompt — effort 是参数层的控制，prompt 仍然是核心
