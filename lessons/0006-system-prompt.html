System Prompt 设计

Lesson 6 · 构建高质量系统提示词 · 约 25 分钟

前面几课关注的是用户消息层面的技巧。本课上升到系统提示词（System Prompt）层面—— 这是你在 API 或应用层面设置的基础指令，它贯穿整个对话，定义 AI 的身份、行为和边界。

► 官方参考：Anthropic — Give Claude a role · Context Engineering

Anthropic 在 Context Engineering 文章中指出：一个 200 token 的系统提示词往往比 2000 token 的效果更好。精简才是王道。

System Prompt vs User Prompt

System PromptUser Prompt
作用设置身份、行为边界、全局规则每次交互的具体任务
频率一次设定，贯穿对话每次对话都可能不同
风格简洁、精确、持久灵活、任务特定
典型内容角色定义、行为准则、输出默认值具体问题、文档、示例
写不好后果整个对话偏航单次回答不准
经验法则：把所有"这个 AI 应该是什么样的"信息放 System Prompt。把所有"这次我要它做什么"的信息放 User Prompt。

优秀 System Prompt 的四要素

1. 角色定义

一句话说清楚"你是谁"——不是客气话，而是精准的身份描述。

你是一名资深数据工程师，精通 ETL 管道设计和大规模数据处理。
你的输出始终以可运行为优先，次优方案会在备注中标注。

2. 行为约束

明确什么能做、什么不能做。记住 L4 的"正面指令"原则：

行为准则：
- 在给出代码前，先用 <approach> 简要说明方案思路
- 代码必须包含完整的类型注解
- 对任何不确定的 API 行为，标注 TODO 而不是虚构
- 默认用 Python，只有用户明确要求时才用其他语言

3. 输出默认值

设定输出的默认风格和格式：

输出规范：
- 不要开场白，直接给出核心内容
- 代码块标注语言（```python）
- 复杂逻辑必须附测试用例
- 多方案对比时用表格

4. 上下文（为什么）

告诉 AI 这个 System Prompt 背后的原因——它可以在你没想到的场景中做出正确判断。

上下文：
你的回答会直接展示给非技术客户。他们不需要了解实现细节，
只需要知道方案能解决他们的问题、需要多少预算、多长时间。
✏️ 测验 1：以下哪个写法更适合 System Prompt？

黄金比例：最小充分原则

Anthropic 观察：很多团队把 system prompt 从 100 字堆到 2000 字，但效果没有提升。每加一条规则，都是潜在的冲突点。从最小版本开始测试，只加你看到有问题的规则。

一个常见错误是把 system prompt 写成「保险条款」——试图穷举所有可能的规则。 Anthropic 推荐相反的路径：

  1. 从最小版本开始：只写角色定义 + 一条关键约束
  2. 观察失败模式：用真实场景测试，记录 AI 在哪些地方不符合预期
  3. 精准修补：每发现一个问题，加一条对应规则
  4. 定期裁剪：如果某条规则从未触发过，删除它

示例演化：

// 版本 1（最小）
你是一名 Python 后端工程师。

// 版本 2（修复了"不写测试"的问题）
你是一名 Python 后端工程师。所有代码必须附测试用例。

// 版本 3（修复了"虚构 API"的问题）
你是一名 Python 后端工程师。所有代码必须附测试用例。
对不确定的 API 行为，标注 // TODO 而不是虚构。

特殊情况：过度主动 vs 过度保守

Claude 新模型默认更主动。如果你想要更保守的行为，显式声明：

除非用户明确要求修改文件，否则默认提供信息和建议。
当用户意图不明确时，先澄清再行动。

如果你想要更主动的行为：

默认情况下，直接实施修改而不是只给建议。
如果用户意图不明确，推断最有用且可逆的操作并执行。
✍️ 练习：为一个客服助手写 System Prompt

你需要为一个电商客服 AI 设计 system prompt。要求：友好但有边界、快速识别退换货请求、不虚构政策、对超出范围的请求转人工。

提示：包含四要素——角色、约束、默认值、上下文（为什么）。
你是一名电商客服助手，专为 [XX商城] 提供客户支持。

角色：
亲和、高效的客服代表。用简短句子回复，不啰嗦。

行为准则：
- 退换货请求：立即识别并引导到退换货流程，不讨论其他话题
- 政策边界：只回答[XX商城]官方政策中明确涵盖的问题
- 超出范围：对超出政策范围的问题，说"我需要帮你转接人工客服"并结束
- 严禁虚构政策——不确定就说不知道

上下文：
你的回复将直接展示给客户。他们正在焦急等待帮助。
快速、准确的回答比"全面但啰嗦"更有价值。

总结

  1. System Prompt 定义角色和行为，User Prompt 描述具体任务
  2. 四要素：角色定义 + 行为约束 + 输出默认值 + 上下文
  3. 最小充分：从小开始，观察失败，精准修补
  4. 主动 vs 保守：显式声明你期望的行为倾向
  5. 正面指令：说"要什么"比"不要什么"效果更好
