子代理编排

Lesson 12 · 多 Agent 协作模式 · 约 25 分钟

Claude 4.6+ 具有原生子代理编排能力——它能识别任务何时需要委派给专门子代理， 并自动协调它们的工作。

本课教你如何设计 prompt 来最大化多 Agent 协作的效率， 以及何时不该使用子代理。

► 官方参考：Anthropic — Subagent orchestration

Anthropic 警告：Claude Opus 4.6 有"过度使用子代理"的倾向。它可能为简单的 grep 操作启动一个子代理。需要明确约束。

什么时候用子代理？

适合子代理不适合
任务可以并行执行简单的顺序操作
需要隔离的上下文单文件编辑
独立的工作流，不共享状态需要保持上下文连续的任务
不同任务需要不同专业角色你（主 Agent）就能做的事

控制子代理行为

如果 Claude 过度使用子代理，添加约束：

<subagent_guidance>
只在以下情况下使用子代理：
1. 任务可以并行执行且互不依赖
2. 需要独立的上下文环境
3. 任务涉及完全不同的领域

对于以下情况，直接工作而不是委派：
- 简单的文件读写
- 单文件编辑
- 只需要上下文的代码搜索
- 任何你能直接一步完成的操作
</subagent_guidance>

子代理编排的 Prompt 设计

当需要子代理时，确保工具定义清晰：

// 子代理工具定义
{
  "name": "delegate_to_subagent",
  "description": "将独立子任务委派给专门子代理。
  只在任务可以隔离执行且不需要与当前上下文共享状态时使用。
  子代理有自己独立的上下文环境。",
  "parameters": {
    "task": {
      "type": "string",
      "description": "子代理需要完成的详细任务描述。
      必须包含：目标、输入数据位置、输出格式要求。"
    },
    "reason": {
      "type": "string",
      "description": "为什么这个任务需要委派而不是直接完成"
    }
  }
}

实战模式：并行研究

最常见的子代理场景——并行研究多个独立方向：

主 Agent 规划 → 子代理 A
竞品分析 → 汇总整合
主 Agent 规划 → 子代理 B
技术调研 → 汇总整合 
请调研以下几个独立方向，每个方向委派一个子代理并行执行：

1. 竞品 A 的最新功能（来源：竞品官网）
2. 技术方案 X 的社区评价（来源：GitHub、Reddit）
3. 相关开源工具对比（来源：各工具文档）

每个子代理的输出格式：
- 3 个关键发现
- 1 个推荐结论
- 信息来源链接

所有子代理完成后，汇总分析并给出最终建议。

避免过度编排

你需要警惕的信号：

  • AI 为简单操作（如 grep）启动子代理
  • 多个子代理做重复的工作
  • 子代理需要频繁与主 Agent 同步

修复方法——在 system prompt 中加入约束：

子代理适合并行执行独立任务。对于简单任务、顺序操作、
单文件编辑或需要跨步骤保持上下文的任务，直接工作。
✍️ 练习：设计一个多 Agent 研究任务

你需要比较 3 个前端框架（React、Vue、Svelte）的最新版本特性。请设计一个 prompt，合理使用子代理并行调研。

提示：每个框架委派一个子代理，各自输出结构一致的结果，最后汇总对比。
请并行调研 3 个前端框架的最新版本特性。每个框架委派一个子代理。

每个子代理的任务：
1. 调研 [框架名] 最新主要版本的 3 个关键新特性
2. 评估性能变化
3. 评估迁移成本
4. 给出采用建议

输出格式（每个子代理独立输出）：
- 框架: [名称]
- 版本: [版本号]
- 新特性: [3 个]
- 性能影响: [提升/下降/不变]
- 迁移成本: [高/中/低]
- 推荐: [是/否]

调研完成后，汇总对比三个框架，给出最终推荐。

跨模型适用性

子代理编排是Claude 特有的高级功能，但思想通用：GPT-4o 可以通过并行 function calling 实现类似效果，Gemini 通过 distribute 工具。国产模型目前对子代理模式支持较弱，建议等待模型更新或使用中间编排层。

总结

  1. 子代理 = 并行执行独立任务的隔离上下文
  2. 什么时候用：独立、可并行、不共享状态的任务
  3. 什么时候不用：简单操作、顺序工作、需保持上下文的任务
  4. 约束过度使用：在 system prompt 中明确子代理的使用边界
  5. 工具定义清晰：子代理工具需要详细的 task 参数描述
